# Self-Incrimination — Overnight Experiment Results (MSH-206)

**Run:** 2026-06-15, fully autonomous. **Host:** bleak-mushroom-dove, GPUs 0+1 (now released).
**TL;DR:** Training Qwen3-4B to append `report_scheming()` when it is *confidently wrong*
**works and generalizes to held-out prompts**, and exposing that self-report to a regular LLM
monitor **substantially improves the monitor's ability to detect task failure**.

## Headline result — no-CoT runs (held-out 100 prompts, disjoint from training)

| LR | self-report P / R / F1 | monitor AUROC: baseline → report-aware | **lift** |
|------|------|------|------|
| 5e-6 | 0.918 / 0.848 / 0.882 | 0.636 → 0.795 | +0.159 |
| 1e-5 | 0.870 / 0.882 / 0.876 | 0.651 → 0.758 | +0.107 |
| **2e-5 (selected)** | 0.909 / 0.870 / **0.889** | 0.635 → **0.801** | **+0.166** |

- **Self-report quality:** the SFT'd model emits `report_scheming()` iff it actually violated the
  task constraint with **F1 ≈ 0.88** on *unseen* prompts — the learned policy generalizes, it did
  not merely memorize training answers (train loss collapsed to ~0, but eval is on disjoint prompts).
- **Monitor lift:** a regular LLM monitor (Qwen3-4B-I) scoring task compliance 0–10 detects
  constraint-violations better when it can see the self-report. AUROC of (1−score) predicting
  ground-truth violation rises **+0.11 to +0.17** across LRs. The report-aware monitor pushes the
  mean score on violated rows from ~0.73 down to ~0.43–0.49 while keeping satisfied rows high.
- **Robust across all three LRs** — the effect is not an artifact of one config.

**Selected model:** `outputs/self_incrimination/sft_lr2e-5/weights/step_50` (best F1 + best lift;
5e-6 is essentially tied and had 0 judge errors if you prefer the cleanest run).

## What was built / run
- **Data (R1):** 400 training rollouts (`outputs/self_incrimination/train400.jsonl`) generated by
  Qwen3-4B on ift tasks, labeled by rule-based `ConstraintScorer` + hedging `ConfidenceScorer`
  (Qwen3-4B-I). 248 malicious (confidently-wrong → sentinel) / 152 honest. Exported to HF parquet
  (`outputs/self_incrimination/sft_dataset`). CoT stripped from training targets (deliberate).
- **Training (R2):** full FT of `/shared/matan/models/Qwen3-4B`, 2-GPU FSDP via `uv run sft`,
  50 steps (~2 epochs), LR sweep {5e-6, 1e-5, 2e-5}. ~4 min/run. Loss collapsed to ~0 (memorized
  the 400 examples) → model selection moved from train-loss to held-out behavioral eval.
- **Eval (R3):** held-out 100 (start_index=400, disjoint from train), via the C4 eval harness
  (`_m_self_incrimination.eval`): baseline monitor (self-report stripped) vs report-aware monitor
  (`REPORT_SCHEMING_MONITOR_POSTFIX` appended, mirroring `_m_confessions`).

## Caveats / notes for review
1. **Train loss → 0 (memorization).** Expected with 400 examples × 2 epochs full-FT. Held-out eval
   on *disjoint prompts* is the real test and it passed; but if you want a cleaner generalization
   story, a shorter schedule (~0.5 epoch) or more training data would reduce memorization. Not done
   (5e-6 already met the objective); flagging as the obvious next knob.
2. **1 judge flake** in the 2e-5 run: Qwen3-4B-I emitted a degenerate repetitive output on one
   monitor_baseline call → unparseable → harness scored it 0.0 (1/100). Minor; doesn't change the
   ranking. A retry/guard on the monitor judge would remove it.
3. **Serve race (fixed).** First eval loop lost 1e-5/2e-5 to a GPU-release race (orphaned vLLM
   `EngineCore` held VRAM; only `sleep 5` between serves). Hardened `si_serve_and_eval.sh` to kill
   EngineCore and wait for GPU drain before/after serving; re-ran both cleanly.
4. **Selection criterion** changed from train-loss (uninformative — all collapsed) to behavioral
   held-out metrics. Logged in tweak-log.
